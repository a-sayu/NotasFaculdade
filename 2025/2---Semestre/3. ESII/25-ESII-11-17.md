CMM Nível 2 - Repetível

Etá em vigor um sistema de gerenciamento de projeto. O processo de construção de software é uma caixa preta, mas com nível 2 repetível ele continua sendo uma série de caixas pretas, mas mais controlável.

A experiência com projetos similares é possível repetir, ocorrendo uma dependência das experiências anteriores, e no desenvolvimento de novos tipos, ocorre um desequilíbrio, não representando o que é necessário para um novo tipo de software.

CMM Nível 3 - Definido

O processo padrão, que ainda é uma caixa preta, é possível enxergar a sequência das tarefas dentro de cada caixa preta, entendendo as funções e responsabilidades de cada um.

Há um processo de engenharia e um padrão para a empresa, as saídas de uma atividade fluem naturalmente para as entradas. Tem-se um processo padrão básico, onde é possível adaptá-lo para o que for necessário, exisitndo um grupo para o processo de software responsável por facilitar ele, existindo um programa de treinamento, tendo o poder de realizar conforme o que receber de artefato, não havendo preocupação de entender o que precisa ser feito.

CMM Nível 4 - Gerenciado

E então o produto e processo são gerenciados quantitativamente. Gerando medidas e estimativas.

Organizações previsíveis.  A organização come;ca a aplicar métricas de controle de qualidade para aumentar a qualidade e a produtividade do software.

CMM Nível 5 - Otimizado

Melhoria Contínua, caracterizado pela existência de processos de software com cont;inua melhoria, onde melhorias podem ser avaliadas e generalizadas para o processo, atinginfo um ambiente de excelência em engenharia de software.

Processos de níveis mais altos podem ser realizados até mesmo por organizações de nível 1, mas se não fizer o básico, ela não sai do nível 1. A competência é feita passo a passo, para sair dos estágio.
